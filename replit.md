# Life Assistant

## Overview

This is a "Life Assistant" web application that uses Claude (Anthropic's API) to provide AI-powered chat functionality. The app features an onboarding questionnaire flow that collects user information, generates a personalized "mirror" profile, and then transitions into a chat interface where users can interact with an AI assistant that has context about them.

The project has a hybrid architecture: a simple Express server handles API proxying and static file serving, while the frontend is a single-page vanilla HTML/CSS/JS application. There's also a more complex full-stack setup (React + Express + Drizzle + PostgreSQL) scaffolded in `package.json` but the active application currently runs from `server.js` and `public/index.html`.

## User Preferences

Preferred communication style: Simple, everyday language.

## System Architecture

### Current Active Architecture

- **Server**: A simple Express.js server (`server.js`) running on port 5000 that:
  - Serves static files from the `public/` directory
  - Provides a `/api/chat` POST endpoint that proxies requests to the Anthropic Claude API
  - Uses a catch-all route to serve `index.html` for SPA behavior

- **Frontend**: A single-file vanilla HTML/CSS/JS application (`public/index.html`) with:
  - A multi-step onboarding questionnaire (name, age, goals, challenges, etc.)
  - Progress tracking through the questionnaire
  - A "mirror" summary generated by Claude based on user responses
  - A persistent chat interface after onboarding
  - Dark theme with purple/pink gradient accents, using Playfair Display and DM Sans fonts

- **AI Integration**: Direct API calls to Anthropic's Claude API (`claude-sonnet-4-20250514`) via the Express proxy endpoint. The API key is stored as an environment variable `ANTHROPIC_API_KEY`.

- **Entry Point**: `server/index.ts` simply imports `server.js`, so running `npm run dev` (which uses tsx to run `server/index.ts`) effectively starts the Express server defined in `server.js`.

### Scaffolded But Not Actively Used Architecture

The `package.json` includes dependencies for a much more complex stack that isn't currently wired up:
- **React frontend** with Vite, TanStack React Query, Wouter (routing), React Hook Form
- **UI components**: Full shadcn/ui component library (Radix UI primitives, Tailwind CSS, class-variance-authority)
- **Database**: Drizzle ORM with PostgreSQL (`connect-pg-simple` for sessions, `@neondatabase/serverless` or standard pg)
- **Schema validation**: Zod + drizzle-zod
- **Authentication**: express-session with Passport.js (passport-local)

If expanding the app, these dependencies are available and the conventional structure would be:
- `client/` - React frontend code
- `server/` - Express API server
- `shared/` - Shared types/schemas (Drizzle schema, Zod validators)
- `db/` - Database configuration

### Key Design Decisions

1. **API Key Proxying**: The Express server acts as a proxy so the Anthropic API key stays server-side and is never exposed to the browser.
2. **Single-file Frontend**: The entire UI lives in one HTML file with embedded CSS and JS — fast to iterate but harder to maintain at scale.
3. **No Database Currently**: All state is client-side only. User profile data from the questionnaire is held in memory/localStorage during the session.

## External Dependencies

- **Anthropic Claude API**: Core AI functionality. Requires `ANTHROPIC_API_KEY` environment variable. Uses the messages endpoint (`https://api.anthropic.com/v1/messages`) with model `claude-sonnet-4-20250514`.
- **Google Fonts**: Loads Playfair Display and DM Sans font families from Google Fonts CDN.
- **PostgreSQL** (scaffolded, not active): The `package.json` includes `connect-pg-simple`, `drizzle-orm`, `@neondatabase/serverless`, and related packages. When a database is needed, use the `DATABASE_URL` environment variable and Drizzle ORM with `drizzle-kit push` for schema migrations.
- **Express.js**: Web server framework, running on port 5000, bound to `0.0.0.0`.
