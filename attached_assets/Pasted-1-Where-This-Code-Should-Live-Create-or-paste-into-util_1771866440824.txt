✅ 2. MOST IMPORTANT STEP
Call Order (This Is What Actually Reduces Cost)

At the start of every chat request, your route must follow this order.

Inside your /chat endpoint:

app.post("/chat", async (req, res) => {

  const user = getUser(req);
  const session = getSession(req);

  // ✅ STEP 1 — daily reset
  resetDailyUsage(user);

  // ✅ STEP 2 — session timeout protection
  checkSessionTimeout(session);

  // ✅ STEP 3 — truncate input
  const { text, truncated } =
      truncateInput(req.body.message);

  // ✅ STEP 4 — depth detection
  const depth = getDepthScore(text);

  // ✅ STEP 5 — budget + model routing
  const budgetState =
      checkBudget(user, depth);

  const model = budgetState.model;
  const efficiencyMode =
      budgetState.efficiencyMode || false;

  // ✅ STEP 6 — output limit
  const max_tokens =
      maxTokens(depth, efficiencyMode);

👉 THIS sequence is what activates savings.

If order changes → cost protection weakens.

✅ 3. You Still Need ONE Helper

Your depth scoring references:

hasDecisionLanguage(m)

You must define it or Node will crash.

Add:

function hasDecisionLanguage(text) {
  return (
    text.includes("should i") ||
    text.includes("what should") ||
    text.includes("decide") ||
    text.includes("choose") ||
    text.includes("not sure") ||
    text.includes("i wonder")
  );
}

Without this → runtime error.
