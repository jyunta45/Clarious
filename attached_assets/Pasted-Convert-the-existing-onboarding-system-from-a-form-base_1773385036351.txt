IMPORTANT IMPLEMENTATION RULE

The onboardingEngine must control the onboarding structure.

The AI model should ONLY generate natural language responses and questions based on the current onboarding category provided by the engine.

The engine — not the AI — decides when to:

- move to the next category  
- pause onboarding  
- mark progress  
- resume onboarding  
- complete onboarding  

This ensures the onboarding flow remains structured while the conversation feels natural.

While onboardingComplete === false, Clarus must prioritize onboarding guidance over normal conversation topics.

CONTEXT

My app currently shows new users a list of questions to answer before their first conversation begins.

I want to replace this with a natural conversation where Clarus guides the user through the same questions organically — one at a time — through dialogue.

The user should never see a list.

They should feel like they are having a first conversation — not filling out a form.

PART 2 — Conversational Onboarding Engine

Create a new file:

onboardingEngine.js

This engine controls onboarding progression.

Responsibilities:

- Track the active category  
- Track exchangeCount  
- Decide when to move forward  
- Handle pause/resume  
- Trigger memory extraction  
- Mark onboarding completion  

The AI only generates conversational language.

The engine controls all structural decisions.

---

ONBOARDING STATE USAGE

For every incoming user message during onboarding:

1. Read onboardingState.currentCategory  
2. Read onboardingState.exchangeCount  
3. Increment exchangeCount by 1  
4. Update onboardingState.lastUserMessageId

Category completion occurs ONLY when the engine decides to move forward.

Completion conditions:

- exchangeCount reaches 3  
OR
- the engine determines the user already covered enough information

When a category completes:

- onboardingProgress[currentCategory] = true
- onboardingProgress.lastCategoryDiscussed = currentCategory
- onboardingState.currentCategory = nextCategory
- onboardingState.exchangeCount = 0

Save updated onboardingState and onboardingProgress.

QUESTION FLOW LOGIC

The onboardingEngine progresses sequentially through the 8 categories.

After each user response:

1. Clarus acknowledges the user's answer in one warm sentence.

2. Call the existing Haiku memory extraction.

3. Extracted data maps to existing memory fields.

4. Engine decides whether to continue within category or move forward.

When moving forward, the engine generates a natural bridge to the next category.

Example bridge:

"That helps me understand your day better. I'm curious about something a little deeper — if your life looked exactly the way you wanted it to, what would you feel most proud of?"

PAUSE AND RESUME

- Save onboardingState
- Save onboardingProgress

Clarus responds:

"No problem. Everything is saved. Come back anytime and we'll pick up where we left off."

When user returns and onboardingComplete === false:

- onboardingComplete = true
- onboardingState = {}

Remaining understanding is gathered gradually through conversation.
